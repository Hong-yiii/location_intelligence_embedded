Implemented services

The device discovers services on the GAP Peripheral and clones the database locally by using the dynamic GATT database feature. The application acts afterward as a proxy for that device and will respond to service discovery queries made by the remote GAP Central device. The application supports all services defined by the Bluetooth SIG. Connection with the GAP Peripheral that contains the database to be cloned is made based on the specified service in the following macro from app.h: gAppProxySelectService_d. By default, the application checks for GAP Peripherals that support the Heart Rate Service (0x180D).
