User interface

After flashing the beacon, the sensor is put in deep sleep (all LEDs are off). To flash the board in case the beacon is put in deep sleep, press ADVSW or RESET. Also, any attached debugger loses its connection. The application default configuration enables low power, which disables LED support. The user can manually change the configuration and enable the LED support, otherwise all subsequent LED behavior references shall be ignored.

Extended advertising is supported only on KW38 platforms. For all the other platforms, the application can only be configured to use legacy advertising (gBeaconAE_c set to 0). In this configuration, first press of the advertising switch starts the legacy advertising and the second press stops it.

On the KW38 boards configured to use extended advertising also (gBeaconAE_c set to 1), the state machine has few more steps:
  • The first press starts legacy advertising.
  • The second press stops legacy advertising and starts extended advertising.
  • The third press stops extended advertising carrying data and then starts extended advertising without data and periodic advertising.
  • The fourth press stops periodic advertising and extended advertising without data and starts legacy advertising and extended advertising.
  • The fifth press stops them all.
For FRDM-KW38 boards, the default enabled low-power modes are 8 and 5. This means that the MCU is put into VLLS2 or VLLS3 stop mode. In this mode, the LED module operation is not implemented. When low power is disabled, LEDs show the advertising status (if they are on the board) as follows:
  • LED2 is ON while legacy advertising is ON,
  • LED3 is ON while extended advertising is ON, and
  • LED4 is ON while periodic advertising is ON.
